Patch 1.2:

Oновлення, які були зроблені в коді: 
Оновлення функції find_tickets: Зробив так, щоб можна було вибрати кілька квитків одночасно, без видалення інших доступних місць. Додана підтримка для перевірки списку місць на наявність і створення квитків тільки якщо всі запитувані місця доступні.
Оновлення інтерфейсу: Додав поле для введення кількох номерів місць через кому, щоб можна було вибрати кілька місць одночасно.
Додав можливість створення нових поїздів: Додав функції та елементи інтерфейсу для створення нових поїздів з визначеною кількістю вагонів та місць у кожному вагоні.
Функція find_tickets: Тепер функція приймає список номерів місць і перевіряє кожне місце на наявність. Якщо всі запитувані місця доступні, вона створює квитки і оновлює список доступних місць.
Графічний інтерфейс: Тепер включає поле для введення кількох номерів місць (через кому).

Patch 1.3:

Зміни, внесені в код, включають:

1. Створення комірки для відображення вже куплених білетів:
    - Додано новий Label (self.lbl_booked_tickets) для відображення заголовку "Booked Tickets:".
    - Додано новий Listbox (self.lst_booked_tickets), який відображає вже куплені білети на введений номер потяга. Цей Listbox показує всі зайняті місця у вибраному потязі після введення номера потяга та натискання кнопки "Find Tickets".

2. Розширення комірки для відображення замовлень:
    - Збільшено розмір Listbox (self.lst_orders), що використовується для відображення поточних замовлень користувача. Тепер його ширина становить 80, а висота – 10, що дозволяє вивести більше інформації без необхідності прокручування.

3. Форматування відображення інформації про замовлення:
    - Поліпшено спосіб відображення деталей замовлення в Listbox. Тепер кожне замовлення включає дату замовлення, кількість білетів, номер потяга, номер вагона, номери місць і статус замовлення (виконано чи ні). Це робить інформацію про замовлення більш читабельною та зручною для розуміння.

Patch 1.5:

У цьому коді були внесені наступні зміни:

Додані нові віджети та функції:

Додані нові віджети для кнопок "Profile" та "Logout".
Додано новий метод show_profile, який відкриває нове вікно профілю користувача, де можна переглянути інформацію про користувача та вибрати аватар.
Додано метод select_avatar, який дозволяє користувачеві вибрати аватар зі своїх файлів.

Перевірка авторизації:

Додано метод check_logged_in, який перевіряє, чи користувач увійшов у систему, та відповідно відображає або приховує елементи інтерфейсу.
Відповідно змінено методи login та logout, щоб використовувати новий метод check_logged_in.
Збереження аватара:

Додано збереження шляху до аватара в JSON-файл користувачів, щоб аватар залишався після перезапуску програми.

Переміщення функціоналу вибору аватара:

Функціонал вибору аватара було переміщено з процесу реєстрації до профілю користувача. Тепер користувач може обрати аватар лише після входу в систему.

Patch 1.6:

Зміни, які були внесені у код:

1.Клас User:

Додано атрибут order_history для зберігання історії замовлень користувача.
Додано методи add_order і complete_order для додавання та завершення замовлень користувача.

2.Клас Order:

Додано атрибут completed для відстеження стану завершення замовлення.
Додано метод complete_order для позначення замовлення як завершене.

3.Клас RailwaySystem:

Додано метод register_user, який додає нового користувача до системи.
Додано метод save_user_data, який зберігає дані користувачів у файлі "users.json".
Додано метод load_user_data, який завантажує дані користувачів з файлу "users.json".
Додано атрибут logged_in_user для зберігання поточного залогіненого користувача.

4.Клас Application:

Додано кнопку "Logout" для виходу з облікового запису користувача.
Додано кнопку "Profile" для відображення профілю користувача з можливістю перегляду історії замовлень та вибору аватарки.
Додано функціонал для реєстрації, входу, виходу користувача, відображення профілю, вибору аватарки, пошуку квитків та завершення замовлення.
